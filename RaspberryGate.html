About

In the contemporary era of rapid societal and technological changes, global incorporation of technology, particularly employee monitoring tools, is recognized for optimizing operations and enchancing accountability. The study focuses on exploring the Raspberry Pi's adaptibility and economic viability for improving office productivity and security. Embracing IoT developments, the research aims to address challenges in office automation, particularly the security of sensitive areas like entrances. RFID technology, with its cost-effective and versatile applications, plays a key role in securing office spaces and ensuring efficient automation.

The integration of RFID, Raspberry Pi, and IoT is positioned as a transformative approach, offering significant potential to revolutionize office productivity monitoring, security, and reporting. The overarching goal is to address security concerns, enhance resource management, and streamline administrative tasks for more efficient and effective office operations.

User Documentation

Raspberry Gate User Manual

1. Introduction:

    Welcome to Raspberry Gate, a smart attendance system designed to simplify the process of tracking attendance using RFID cards. This user manual will guide you through the setup, usage, and maintenance of the Raspberry Gate system.

2. Components:

    a.Raspberry Pi 3B: The central computing unit that runs the Raspberry Gate script and manages the entire system. It processes data from the RFID card reader, controls the LCD display, and handles attendance logging and exporting.

    b.RFID Card Reader: Reads data from RFID (Radio-Frequency Identification) cards. Each card is associated with a unique identifier that is used to authenticate users. When a card is tapped, the RFID card reader sends the card's unique identifier to the Raspberry Pi for processing.

    c.I2C-enabled LCD Display: Provides a visual interface for users and displays relevant information such as user names, access status (granted/denied), and timestamps. It enhances the user experience by communicating real-time interactions and system responses.

    d.Internet Connection: Essential for connecting the Raspberry Pi to the internet. The internet connection is crucial for two main purposes: authenticating with Google Drive for attendance export and ensuring the system has accurate timekeeping for timestamping attendance entries.

    e.Google Drive: Used as a cloud storage platform for exporting attendance data. The system utilizes the PyDrive library to authenticate with Google Drive and upload attendance sheets periodically or manually triggered.

    f.Desktop (File Storage): The system creates a folder named "Raspberrygate" on the user's desktop (or a specified location) to store exported attendance sheets as text files. This local storage ensures a backup of attendance data in addition to the cloud-based storage on Google Drive.

3. Assembly:

    a.Circuit connection:

      i.RFID IC ID Card Reader:

        -Plug the USB connector of the RFID IC ID Card Reader into an available USB port on the Raspberry Pi.

        -Ensure that the RFID IC ID Card Reader is securely connected.

      ii.I2C LCD Display:

        -Connect the SDA pin of the I2C LCD Display to the SDA GPIO pin on the Raspberry Pi.

        -Connect the SCL pin of the I2C LCD Display to the SCL GPIO pin on the Raspberry Pi.

        -Connect the 5V pin of the I2C LCD Display to a 5V GPIO pin on the Raspberry Pi.

        -Connect the GND (ground) pin of the I2C LCD Display to a GND GPIO pin on the Raspberry Pi.

      iii.Monitor:

        If using an external monitor for the Raspberry Pi, connect the monitor to the appropriate port (HDMI, VGA, etc.).

      iiii.Power On:

        Once both the RFID IC ID Card Reader and I2C LCD Display are connected, power on the Raspberry Pi.

4. Software Setup:

    To set up the Raspberry Gate software, open the Raspberry Pi terminal and install essential libraries using "pip install pydrive RPLCD." Download the script from the source or repository. Run the system by navigating to the script's directory and executing "python raspberry_gate_script.py" in the terminal. This activates RFID card authentication and attendance logging.

5. Usage:

    To log in or out, simply tap your RFID card near the reader. The LCD displays your name, access status, and the time. "Access Granted" confirms successful login, and your attendance is logged. If the card is invalid, "Access Denied" appears.

    For attendance export, it happens automatically every 10 minutes to Google Drive. You can also trigger an instant export by tapping the card specified as "export_trigger_card."

6. Maintenance:

    To maintain optimal performance, regularly check hardware connections, keep components free from dust, and handle hardware with care. Periodic checks ensure secure connections, cleaning prevents malfunctions due to debris, and gentle handling prolongs the life of the system.

7. Troubleshooting:

    If you encounter "Access Denied," check for a valid RFID card and typos in the card number. For export issues, ensure a stable Raspberry Pi internet connection and verify Google Drive authentication.
